What is Open Science?

Data Sharing

The expansion of information technology in recent decades has enabled a new phase of scientific research in which the results of studies may be shared in ways that were never possible before. This phase has been sometimes called Open Science, which indicates a practice where the data collected from scientific research is shared as widely and as openly as possible. That said, the importance of this to the scientific discipline goes beyond just open access. Indeed, it has begun to transform the discipline itself by ushering in a new standard of transparency and reproducibility to the scientific method.

The Scientific Method

The scientific method can be briefly summarized as:

  1. The collection of data of some kind, followed by
  2. The development of hypotheses to explain this data, and then
  3. The testing of these hypotheses using experiments to gather new data that may contradict these above hypotheses.

However, a single experiment whose results conform to a hypothesis does not affirm that hypothesis, as

  • every experiment is subject to errors that can invalidate conclusions drawn from its data, and
  • no single experiment can collect all the data necessary to test a hypothesis.

Due to this, the scientific discipline must also make its methodologies

  • transparent, so that experimental errors may be identified if present, and
  • reproducible, so that further experimental data may be collected that either corroborates or invalidates the hypotheses posed in the original experiment.

Innovations of Open Science

While science has always strived to solve the problems of transparency and reproducibility in its discipline through documentary practices, peer-review, consortial studies, and more, Open Science has established a new frontier to address these issues. Open Science does so by establishing both the means and standards for sharing scientific data so that not only are the methods and results of studies widely available, but also so that they may be commonly understood and reinterpreted by the entire scientific community. Thus, Open Science makes it possible to:

  • check for errors in the methods used by researchers to produce their data,
  • conduct new experiments on new data sets with the same methodologies, and
  • reanalyze and compare existing datasets to develop new hypotheses and corroborate/invalidate old hypotheses using massive datasets and analysis software

Open NeuroImaging Databases

OpenNeuro

  • About
  • Access
  • Licences & Citation

OpenNeuro contains a growing collection of over 500 open MRI datasets sourced from over 17,000 participants, including humans, rats, mice, and other species. Moreover, this data spans multiple imaging modalities beyond MRI, including PET, EEG, iEEG, and MEG. All datasets are within the public domain and use the BIDS format. The datasets can be browsed using a friendly interface that allows for keyword searches as well as by factors such as the number of participants, their diagnosis type, or the type of task being performed during the scan. All datasets come with further information about the study that provide a research context for the data.

For advanced access to OpenNeuro data, see the section of this guide on DataLad for detailed instructions. For simple access, see here.

Go to OpenNeuro in a Google Chrome browser (Firefox and other browsers have limited functionality). To access all features, such as bookmarking, sign up to OpenNeuro using either a Google or an ORCID account.

After you have created your login, use their search interface to browse through all available datasets. Once you have located a dataset that you would like to use, you can download it in its entirety in the BIDS format by clicking on its entry followed by the download button on its page, and then using “Download with your browser” option from those available. Lastly, when your file browser pops-up, select the folder you would like the data to be saved within, then create a new folder with a name for the dataset. Click select and give permission to the browser to view files and save changes. Note that many datasets have large file sizes, so download times can vary.

Note that individual files may also be downloaded or viewed in isolation from the rest of the dataset by browsing the file tree on the main page of the study.

Every OpenNeuro dataset has been licensed to be a part of the Public Domain, meaning that it can be freely used without permission or attribution of the dataset's creators. Nonetheless, it is still important to cite your usage of the dataset in publications, presentations, etc. to inform your audience of the data's original source and to give credit where it is due.

Every dataset includes information from their creator's that specifies how they would like to be acknowledged. You can find this in the right-hand column of the dataset's page under "How to Acknowledge". You may also find shortcuts for including citations in text and BibTex, as well as the dataset's doi.

EBRAINS

  • About
  • Access
  • Licences & Citation

EBRAINS is an innovative neuroscience research infrastructure that comes from the EU’s Human Brain Project, which is a 10-year project that began in 2013 to gather neuroscientific data, establish neuroscientific metadata standards, and create a neuroscience computing and information network that would allow researchers worldwide to access and analyze this data easily. EBRAINS contains over 650 freely accessible neuroimaging datasets, the majority of which are based on human subjects. Special access can also be granted to use the EBRAINS Knowledge Graph, which is a visualization tool that can be used to demonstrate the relationships between various datasets, models, and software – expanding the researcher’s toolkit. This access also grants use of analysis software made available from the Human Brain Project.

First, register for an EBRAINS account using your institutional email. If you do not have an institutional email, then you may apply for access by sending a request from your personal email to support@ebrains.eu.

You can search for datasets by using keywords and by filtering according to imaging modality, species, data accessibility, and methods tags. You may also browse software tools and models.

Once you have located the dataset that you are interested in, click on it. In the pop-up window, click the “Get data” tab, and select the folder that you would like to download. Once you have done this, click “Download dataset”, and the dataset will download as a .zip folder.

Many, but not all, datasets in the EBRAINS database have a CC BY-NC-SA 4.0 licence, meaning that their use must attribute its creator, it must not be used for commercial purposes, and its use in a work must also have a CC BY-NC-SA 4.0 licence, or one that is compatible.

To attribute the dataset, you may use the "How to Cite" tab in the dataset's window on EBRAINS.

ConnectomeDB

  • About
  • Access
  • Licences & Citation

ConnectomeDB collects data from the Human Connectome Project (HCP), which was a major project undertaken in 2009 to understand the connections that can be found between neurons in the brain, creating a network map using MRI and other imaging techniques. The project was led by two consortial groups with over 1000 healthy young participants and has since led to over 20 other projects through the Connectome Coordination Facility that examine other populations. Their website also includes software downloads for the data visualization and preprocessing that is used in the HCP.

To access the Connectome Database, you must first register for an account on their login page. Account creation requires the input of a home institution, such as Western University, as well as an email verification, so make sure to check your spam filter if it has not come through.

After you create an account, you may login to access the available datasets, which consist of the:

  • WU-Minn HCP Data (1200 subjects)
    • 3T MR scans from young healthy adult twins and non-twin siblings using structural T1w and T2w, resting-state fMRI, task-fMRI, and dMRI.
  • WU-Minn HCP Retest Data
    • 46 HCP subjects that were retested with the HCP 3T multimodal imaging and behavioural protocol. Subjects retain their IDs from the above dataset.
  • WU-Minn HCP Lifespan Pilot Data

    • Gathers 3T and 7T T1w, T2w, resting state fMRI, task-fMRI, and dMRI scans of participants across 6 age groups (4-6, 8-9, 14-15, 25-35, 45-55, 65-75).

  • MGH HCP Adult Diffusion
    • T1w, T2w, and dMRI scans of 35 young adult subjects using a 3T scanner with 300 mT/m maximum gradient strength on dMRI scans.

Due to the size of these datasets, ConnectomeDB uses the IBM Aspera Connect plugin for your browser. Be sure to install this on your computer and to enable the Aspera Connect plugin from your browser’s plugins.

While the datasets available on ConnectomeDB are open-access, they require your agreement to various terms of use, which includes acknowledging the funding sources of the HCP and citing the key methods used by the HCP to acquire and process their data. A guide to these publications is found here.

 

Image and Data Archive

  • About
  • Access
  • Licences & Citation

The Image and Data Archive (IDA) provides tools to view, search, and download neuroscience data from your browser. It can also be used by researchers to store their data and control access to it. The IDA contains data from over 150 studies and over 65,000 participants. Studies found in the database vary greatly and include major projects such as the Alzheimer’s Disease Neuroimaging Initiative and the Parkinson’s Progression Marker’s Initiative. A complete list of the projects can be found here.

To begin, create a login using the “Instant Free Signup” button. Access to projects varies from:

  • Open access,
  • Apply to access, and
  • Private

Most major projects fall under the “Apply to access” category. To apply to access a dataset, click “Featured Studies” and click the green “Apply” button on the study you would like to apply to. After agreeing to the terms, access is often granted quite quickly.

To access data from those you have been granted permission to, click on search from the menu. Then, navigate to the “Advanced Search (beta)” tab to being your search. You can filter your search by project, image type, image modality, subject, and more. After conducting your search, select the records that you would like to save from your results by clicking their checkboxes. Once all have been selected, click “Add to Collection”. You will be prompted to either create a new collection or add to an old one. If creating a new one, type a name that describes what data you are collecting.

Once you have done this, click the “Data Collections” tab, and on the left-hand menu under collections, expand “My Collections” and then expand the collection you would like to download. Lastly, select all the data within the collection that you would like to download, and click “1-Click Download”. The files will saved in a .zip folder.

The terms of licences to datasets found on the IDA varies by studies. When you apply for access to a study, you must agree to their terms of use. Within this agreement, there is often a clause that states how to properly attribute the usage of data from that dataset.

For example, the agreement to access the PPMI states:

In manuscripts utilizing PPMI data, I will acknowledge the work by PPMI personnel that went into accumulating the data, as well as funding of the study, and will include language in manuscripts similar to the following:
"Data used in the preparation of this article were obtained from the Parkinson's Progression Markers Initiative (PPMI) database (www.ppmi-info.org/data). For up-to-date information on the study, visit www.ppmi-info.org."

 

It is a good idea to save this information in a table alongside other IDA studies for later use. If you forget to do this, you can always check the webpage of the study or log out of your IDA account and click "Apply" on the study.

StudyForrest

  • About
  • Access
  • Licences & Citation

StudyForrest is a project that began in 2014 and has since taken part in over 50 publications. Its original study gathered a high-resolution fMRI dataset from 20 participants using a high field strength (7 Tesla) to record them while watching the film, Forrest Gump. The principle underlying this experiment was that stimuli such as a feature film offer a naturalistic environment that will elicit brain responses that are akin to the everyday behaviour of the brain. Upon publication, the dataset was made freely available for other researchers to access and contribute to. Since then, other datasets with the same participant group being shown various stimuli have been made available, with participant IDs being consistent across datasets for comparison. Even more, various community members have made available annotations (i.e., descriptions of what is happening in the film, e.g., portrayed emotions of characters) of the Forrest Gump film that can be used in analysis.

To access the StudyForrest data, use DataLad by running the command:

datalad clone https://github.com/psychoinformatics-de/studyforrest-data

For instructions on how to install and use DataLad, see the DataLad section of this guide.

StudyForrest data is licensed under the Public Domain Dedication and License (PDDL), which gives the user of the data virtually limitless freedom over the data's use, including commercial use. Nonetheless, it is still important to practice scholarly citation when using the StudyForrest materials.

StudyForrest recommends that all data used from their project be attributed to the publication that this data is derived from. StudyForrest provides a list of all publications related to their study, those that are relevant to the creation of datasets are in the "Data" category under the column "Type". These studies should be cited according to whatever standard of citation you are using in your research and writing.

BIDS Format

What is BIDS?

Although every neuroscientific study varies in several important ways, there are some important features and information of interest that are common to all neuroimaging studies. The Brain Imaging Data Structure (BIDS) is a recent innovation in neuro-informatics by Gorgolewski et al. (2016) that provides a standard structure for storing and sharing neuroimaging data. This is data structure is important to be familiar with not only because it is the standard used on OpenNeuro, but more importantly because its wide adoption by researchers would enable:

  • Greater collaboration due to a common understanding of data sets across researchers, both within and across research groups
  • A reduction of error due to the misprocessing or misinterpretation of data
  • The expansion of data analysis software due to standardized metadata practices that are made machine-readable, and
  • The use of automated tools to verify data consistency and completeness.

Hierarchical Structure

The BIDS format is a hierarchical folder structure, allowing for simple user-navigation and efficient integration with computer programs. The basic hierarchy is as follows:

  • Project (named to describe the project)
    • Participant (unique label for each participant of the study)
      • Session (a label for each recording session in the study – not required if there is only a single recorded session)
        • Datatype (describes the type of data contained in the folder, such as functional, anatomical, EEG, etc.)

These folders will contain files that represent the metadata that describes the data, as well as the data itself. Typically, files are named in a somewhat standardized fashion to be descriptive of their contents.

Raw and Derived Data

Multiple BIDS-like structures of this kind may be included in a single dataset to share both the raw and processed data. For instance, a dataset may contain a project folder called “masked” which includes masked data, i.e., data that has been filtered to only include the parts of the image that are of interest to the researcher. Derivative datasets are stored in their own folder called “derivatives” to structurally distinguish them from the raw data. Metadata accompanies these datasets as well to describe their content in a generally standardized way, however, new methodologies are always emerging, so there is more flexibility for this category of data.

BIDS Apps

  • General
  • fMRIPrep
  • HCPPipelines

Due to the standardization of the BIDS file format, it has been possible for developers to create a number of applications that can rely on the same set of commands to process large batches of neuroimaging data for various purposes. Applications of this kind are descriptively called BIDS Apps. For personal use, the BIDS Apps are implemented using Docker, which is a tool that can be used to store and run several apps simultaneously in a light-weight manner. You must install docker to use the BIDS Apps.

You can browse all available BIDS Apps here. To download an app, click on its Docker link and run the

docker get creator/app

command included on its page. This will initiate the download in docker. For usage, check out the app's documentation on GitHub by clicking on its name in the BIDS App website, as well as general tips provided by BIDS Apps.

For illustrative purposes, a few notable BIDS Apps are discussed in the following tabs.

fMRIPrep is a BIDS app that is used for the preprocessing of unprocessed task-based and resting-state fMRI data. It processing tools include:

  • motion correction,
  • field unwarping,
  • normalization,
  • bias field correction, and
  • brain extraction.

fMRIPrep is designed to perform equally well on all input datasets, regardless of scanner or correction scans, and moreover, it is simple and transparent due to the BIDS format and the availability of thorough processing reports upon use. Documentation on its usage can be found here, and to run through an example of  fMRIPrep using OpenNeuro datasets, check out its section in Andy’s Brain Book, which includes video demonstrations.

Note that fMRIPrep produces a boilerplate acknowledgement template that should be used in all publications that make us of this app. Note further that fMRIPrep relies on the FreeSurfer tools, which require registration prior to access. To enable your licence after registration, see here in the fMRIPrep documentation.

HCPPipelines is a BIDS App that implements the pipelines used in the Human Connectome Project, as described in Glasser et al. (2013).

Its implementation requires that each subject has both a T1w and a T2w scan.

Documentation on usage can be found here.

Note that for commercial purposes, you should consult the licences listed here, as HCPPipeline uses several packages that are not freely licensed.

DataLad (Data Sharing Tool)

What is DataLad?

DataLad is a data sharing tool that ensures your access to data is the most up to date version possible. It does so by not actually saving data on your computer, but rather references to it, making it a very sophisticated and efficient method of data sharing.That said, DataLad’s sophistication gives way to some complexity that users may have issues overcoming. We attempt to explain DataLad’s installation and use as simply as possible so that you can access the data sooner.

Installation

  • Windows
  • Mac

Step One: Ensure that Python 3 and Git are installed on your computer. These are necessary to use DataLad.

To check if you have these installed, paste and run the following commands in Command Prompt:

python --version

git –-version

If any of these commands returns an error, then that program needs to be installed. Do not use the Windows Store to download these programs, use the links above instead.

Note that when installing Git, you should select the following settings:

  • Enable Use a TrueType font in all console windows
  • Select Git from the command line and also from 3rd-party software
  • Enable file system caching
  • Enable symbolic links

Step Two: You must also install git-annex, which is possible once you have Python 3 and Git installed. To install it, paste and run the following command in Command Prompt:

pip install datalad-installer

Then, paste and run the following command in the administrator mode of Command Prompt (short-cut: Windows Key + R)

datalad-installer git-annex -m datalad/packages

Step Three: To install DataLad, run the following command in Command Prompt:

pip install datalad

Step One: Ensure that Python 3, Git, and git-annex installed. These are all necessary to install and use DataLad.

To check if you have these installed, paste and run the following commands in Terminal:

python3 --version

git --version

git-annex

If any of these commands gives an error, then that program needs to be installed.

Step Two: Check that you have the Command Line Tools for XCode installed by pasting and running the following command in terminal:

xcode-select --install

Next, we are going to install homebrew, which we will use to install DataLad afterwards. To do this, paste and run the following command in Terminal:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Step Three: Install DataLad by pasting and running the following command in Terminal:

brew install datalad

Retrieving Data

  • Cloning Data
  • Getting Data

A very useful aspect of DataLad is that it allows you to create “clones” of datasets, rather than downloading them directly onto your hard drive. Doing so is simple:

Step One: In Terminal (Mac) or Command Prompt (PC), select the folder you would like your data to save to by copying its pathname and pasting and running the following command:

cd pathname

Be sure to replace pathname above with the actual path of the folder.

Step Two: On OpenNeuro, find the address of the dataset by using the search feature. On the datasets page, click on “clone” and from the dropdown, click “copy Github url”.

Step Three: In Terminal (Mac) or Command Prompt (PC), type datalad get, then paste the copied dataset’s address after it. It should look something like:

datalad clone https://github.com/OpenNeuro/ds00XXXX.git

After doing this, run the command. This creates a BIDS formatted structure in your folder named ds00XXXX. This folder contains the study’s metadata and references to the actual data, but, there is no actual imaging data in this folder at this time.

Once you have cloned a dataset, you can retrieve actual imaging data for specific subjects, sessions, and data types.

Step One: Change directories to your data set by running the command:

cd ds00XXXX

Step Two (A): Within your file browser, find the pathname of the file/folder that you would like to download. Once you have done this, you can run the command:

datalad get pathname

Doing this will load all the data in that folder, so that when you click on the files there, they will immediately download.

Step Two (B):  If you would like to download all the data in the entire dataset, run the command:

datalad get .

Step Two (C): To download a specific subset of the data, for example, all the anatomical data for all subjects at all sessions, you can use globbing. For the example above, we would run the command:

datalad get sub-*/ses-*/anat/

Using this procedure allows you to download only the data that you need easily.

NeuroImaging Analysis Software

FSL

  • About
  • Installation
  • Getting Started

The FMRIB Software Library (FSL) is a widely used library of analysis tools for fMRI, MRI, and DTI brain imaging data. This library can be used to streamline:

  • Preprocessing,
  • Analysis of task-based fMRI scans,
  • Analysis of structural MRI scans,
  • Analysis of connectivity, and more

FSL can be installed on Windows, Mac and Linux, though performance is poor on Windows. You can find the instructions for installation on each system here.

FSL offers a two week online course each year in the Fall, although there is a fee for enrolment. If you have the resources and the time availability, consider joining the course.

If you are not able to attend the online course, or you would rather self-study, there is also a self-study option available that comes at no charge. You may access the materials for the course here.

To run through an example of an fMRI analysis using FSL that takes advantage of the OpenNeuro, check out its section in Andy’s Brain Book, which includes video demonstrations.

AFNI

Analysis of Functional NeuroImages (AFNI) is a suite of programs and scripts that can analyze and display fMRI, MRI, and DTI brain imaging data. It is implemented in the programming language C, so advanced users may customize its capabilities.

AFNI can be installed on several operating systems, including Windows, Mac, and Linux, though performance is poor on Windows. You can find instructions for installation on each system here.

You can check out a list of educational resources provided by AFNI here. These resources include the video series AFNI Academy which provides a comprehensive look at using AFNI in a video-format.

To run through an example of an analysis using AFNI that takes advantage of the OpenNeuro database, check out its section in Andy’s Brain Book, which includes video demonstrations.

SPM

  • About
  • Installation
  • Getting Started

Statistical Parametric Mapping (SPM) is a method of hypothesis testing for functional imaging data, and these methods are implemented in the SPM software package. SPM can be used to analyze data from fMRI, PET, SPECT, EEG, and MEG scans.

More information on the uses of Statistical Parametric Mapping can be found in the notes Introduction to SPM by Karl Friston. A more thorough discussion may also be found in the textbook Statistical Parametric Mapping: The Analysis of Functional Brain Images (2007), also by Friston.

SPM is available on Windows, Mac, and Linux. To download SPM, first make sure that you have MATLAB installed on your computer, as SPM is simply a collection of MATLAB functions. To get MATLAB from your institution, click here and enter the requested information for access.

Once you have MATLAB installed on your computer, you may proceed to download SPM. You can select multiple versions, however the most up-to-date version as of this guide is SPM12. To make SPM compatible with MATLAB, follow the instructions for your operating system here.

The manual for SPM12 is freely available, and contains examples of various analyses that may be carried out on datasets available on the SPM website.

Courses on using SPM are given each May and October by the Institute of Neurology and cover multiple imaging modalities. Information on registration as well as materials from past courses can be found here.

A WikiBook for SPM that gives a comprehensive overview of installation, experimental design, data formats, preprocessing, modelling, statistical inference, connectivity analysis, etc. is a good reference for using SPM.

To run through an example of an analysis using SPM that takes advantage of the OpenNeuro database, check out its section in Andy’s Brain Book, which includes video demonstrations.

ComBat

  • About
  • Installation
  • Getting Started

When comparing neuroimaging scans from across different scanners and imaging sites, there are certain factors external to the subjects that can affect the data from the scan. ComBat is a harmonization tool that can reduce these undesired effects, making analyses across multiple scanners and imaging sites feasible.

For more information on the validity of ComBat harmonization for neuroimaging, please see the articles by Fortin et al. (2017, 2018).

ComBat can be implemented in R, Python, and MATLAB. Click below for instructions on the installation of each:

  • R
  • Python
  • MATLAB

Note that each implementation is tested regularly to ensure that they agree with one another.

To run harmonization in each implementation, ComBat requires three inputs:

  • A data array with n rows for the features (voxels, brain regions, etc.) and p columns for participants. Ensure that this data has undergone pre-processing.
  • A batch vector with p entries that specifies the ID of the site, scanner, etc. that is creating differences
  • An optional biological variable array that specifies co-variates that should not be factored out of the analysis, e.g., age, condition, etc.

It outputs an array that matches the size of the data matrix, but this data is harmonized for statistical analysis.
